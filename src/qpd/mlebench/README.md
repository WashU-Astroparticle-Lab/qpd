# QPD MLE-bench datasets

Machine-learning benchmark datasets of complex transmission **S21 = I + iQ**
time traces from a simulated **QPD transmon** dispersively coupled to a notch
readout resonator. Generated by this package (`qpd.mlebench`) from the physics
frozen in [`notebooks/simulation.ipynb`](../../../notebooks/simulation.ipynb).

The task is **event reconstruction**: from a raw I/Q trace, recover the discrete
physical events (quasiparticle tunneling, charge jumps, quasiparticle bursts)
that produced it.

## Two competitions

Both share an identical background condition — electronic white noise, a
quiescent two-state quasiparticle-tunneling telegraph, and discrete
charge-jump events — and are generated at **equal length** in independent
**30 s chunks** sampled at **100 kHz** (3 M samples/chunk).

| competition      | quasiparticle bursts |
|------------------|----------------------|
| `qpd-signal`     | **present** — Poisson onsets (~1 Hz, ≈30 per chunk), each burst injects `Poisson(15)` QP tunnels with an exponentially-modified-Gaussian (EMG) arrival profile |
| `qpd-background` | **absent** — background condition only |

## Reconstruction targets

For each test chunk, predict the ground-truth event list:

1. **Parity flips** — the time of every quasiparticle tunneling event (each
   toggles the charge parity even↔odd). Includes both quiescent telegraph
   switches and, in `qpd-signal`, burst events.
2. **Charge jumps** — the time and the **fractional part** of each offset-charge
   jump, wrapped to (−0.5, 0.5]. The integer part is physically unobservable
   (the Hamiltonian is periodic in `n_g` with period 1), so only the fraction
   is a target.
3. **Quasiparticle bursts** — the **onset time** and the **number of QP
   tunneling events** of each burst (`qpd-signal` only).

## Layout (per competition)

```
qpd-<name>/
  description.md                              # task description for solvers
  metadata.json                              # full physics config + provenance
  manifest.csv                               # per-chunk sample_rate, n_samples, t0, seed, counts
  prepared/
    public/
      train/chunk_XXXX.parquet               # I, Q (float32) — labelled
      test/chunk_XXXX.parquet                # I, Q (float32) — predict these
      train_labels.csv                       # ground truth for train chunks
      sample_submission.csv                  # submission schema (header only)
    private/
      test_labels.csv                        # held-out answer key for grading
```

### Reading a chunk

Each Parquet file has two `float32` columns, `I` and `Q`. The time axis is a
uniform grid reconstructed from `manifest.csv`:

```python
import pandas as pd, numpy as np
m   = pd.read_csv("qpd-signal/manifest.csv").set_index("chunk_id")
iq  = pd.read_parquet("qpd-signal/prepared/public/test/chunk_0357.parquet")
row = m.loc["signal_0357"]
t   = row.t0 + np.arange(row.n_samples) / row.sample_rate_hz   # t0 = 0
s21 = iq["I"].to_numpy() + 1j * iq["Q"].to_numpy()
```

## Labels & submission format

A single long-format CSV, one row per event:

| column       | meaning                                                        |
|--------------|----------------------------------------------------------------|
| `chunk_id`   | e.g. `signal_0042`                                             |
| `event_type` | `flip`, `charge`, or `burst`                                   |
| `t`          | event time within the chunk [s]                               |
| `value`      | `flip`: blank · `charge`: fractional jump · `burst`: QP count  |

Predict no rows for a chunk to predict "no events".

## Grading

Per chunk and per event type (see [`grade.py`](grade.py)):

1. **Optimal matching** — predictions are matched to truth to minimise total
   timing error (Hungarian assignment) within a per-type time tolerance
   (`flip` 0.5 ms, `charge`/`burst` 5 ms). The gated graph is split into
   connected components for speed on dense flip trains.
2. **Continuous soft timing-F1** — each match earns partial credit
   `q = exp(-(Δt / scale)²)` (`scale = tolerance / 2`); then
   `soft_TP = Σq`, `soft_precision = soft_TP / n_pred`,
   `soft_recall = soft_TP / n_truth`. False positives/negatives and loose
   timing all reduce the score continuously.

The overall scalar ∈ [0, 1] (higher is better) is the mean soft timing-F1
across the event types present, lightly penalised by the normalised value
error (circular MAE of the fractional charge; MAE of the burst QP count):

```
overall = mean(timing_F1) × (1 − 0.5 · value_penalty)
```

```python
import pandas as pd
from qpd.mlebench import grade, grade_report
sub = pd.read_csv("submission.csv")
ans = pd.read_csv("qpd-signal/prepared/private/test_labels.csv")
score  = grade(sub, ans)                                  # scalar in [0, 1]
report = grade_report(sub, ans, return_matches=True)      # full breakdown + per-event table
```

## Regenerating

```bash
# Plan only (no data written)
python scripts/generate_mle_dataset.py --dry-run --max-bytes 10e9 --bytes-per-chunk 14e6

# Full run (~357 chunks/dataset, ~9.4 GB, ~13 min)
python scripts/generate_mle_dataset.py --out-dir data/mle_bench \
    --max-bytes 10e9 --bytes-per-chunk 14e6 --seed 0

# Tiny end-to-end smoke test
python scripts/generate_mle_dataset.py --smoke
```

Generation is deterministic in `(master_seed, dataset, chunk_index)`. The byte
budget binds long before the compute budget; `--bytes-per-chunk` calibrates the
planner to the measured ~13.2 MB Snappy-compressed Parquet chunk size so the
budget is filled accurately. The complete frozen physics configuration is
written to each competition's `metadata.json`.
